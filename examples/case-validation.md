# 回归测试说明

发布候选版使用两个不同行业、不同深度的案例回归。真实材料和生成结果不进入公开仓库。

## 公司名-only intake

- 输入：只有公司名称。
- 期望：一次性询问交流对象、补充材料和播客需求；不进行公开检索。

## Standard 材料型案例

- 输入：可公开测试材料、明确交流对象、需要播客。
- 检查：材料页码引用、公开信息交叉验证、人物画像、竞争/KPI表、时间线、矛盾日志、10-15个关键问题。
- 输出：通过工作稿校验和 PDF 校验；逐页 PNG 目视无缺陷。

## Quick 公开信息案例

- 输入：公司名、交流对象、无补充材料、不需要播客。
- 检查：不超过8次检索；3-5页 PDF；8-10个定制问题；证据边界清楚。

## 音频案例

- 有 Key：验证官方 speech Skill 生成 MP3。
- 无 Key Windows：验证离线渲染器生成非空 WAV。
- 两种路径都必须使用重新编写的 5-10 分钟播客稿，而非逐字朗读 Note。

这些测试只证明工作流与质量门槛可执行，不代表测试公司的投资价值。
